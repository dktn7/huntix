# HUNTIX — Sprite Pipeline

> Characters, enemies, and bosses are **2D billboard sprites** on `PlaneGeometry` quads inside a 3D Three.js scene. There are no GLTF files. There is no `AnimationMixer`. Animation is UV frame stepping.

*Last updated April 16, 2026*

---

## Core Concept

The world is 3D (floors, walls, parallax background layers). Everything that moves and fights is a 2D sprite. This gives the Castle Crashers / Hades visual — 3D depth and parallax with crisp 2D character art.

```
Three.js Scene
├── ParallaxLayer (background)   ← PlaneGeometry, static texture
├── ParallaxLayer (midground)    ← PlaneGeometry, slower scroll
├── Floor geometry               ← BoxGeometry or PlaneGeometry
├── HunterSprite (Dabik)         ← PlaneGeometry + MeshBasicMaterial + atlas
├── EnemySprite (Grunt ×5)       ← InstancedMesh PlaneGeometry + atlas
└── ParallaxLayer (foreground)   ← PlaneGeometry, faster scroll
```

Y-sort every frame so sprites closer to the camera (higher Y world value) render on top:
```js
mesh.renderOrder = worldY * 10;  // or: mesh.position.z = -worldY * 0.01
```

## Sprite Atlas Format

All atlases are generated by **TexturePacker** using the **JSON Hash** export format.

### File naming convention

```
assets/sprites/
├── dabik.png           ← atlas image
├── dabik.json          ← TexturePacker JSON Hash
├── benzu.png
├── benzu.json
├── sereisa.png
├── sereisa.json
├── vesol.png
├── vesol.json
├── grunt.png
├── grunt.json
├── ranged.png
├── ranged.json
├── bruiser.png
├── bruiser.json
├── vrael.png           ← Boss
├── vrael.json
└── ...                 ← one PNG + one JSON per character
```

### JSON Hash structure (TexturePacker output)

```json
{
  "frames": {
    "idle_00": {
      "frame": { "x": 0, "y": 0, "w": 64, "h": 96 },
      "sourceSize": { "w": 64, "h": 96 },
      "pivot": { "x": 0.5, "y": 1.0 }
    },
    "idle_01": {
      "frame": { "x": 64, "y": 0, "w": 64, "h": 96 },
      "sourceSize": { "w": 64, "h": 96 },
      "pivot": { "x": 0.5, "y": 1.0 }
    },
    "run_00": { ... },
    "attack_light_00": { ... }
  },
  "meta": {
    "size": { "w": 512, "h": 512 },
    "scale": "1"
  }
}
```

**Pivot:** Always `{ x: 0.5, y: 1.0 }` — bottom-centre. This means the sprite's world position is at its feet, which is correct for Y-sorting.

## Animation State Names

Frame names follow the convention `{state}_{frameIndex}` with zero-padded two-digit index.

### Required states for all characters

| State name | Frames | Notes |
|------------|--------|-------|
| `idle` | 4 | Standing breathe loop |
| `run` | 8 | Movement loop |
| `dodge` | 6 | Dodge/roll — plays once, no loop |
| `hurt` | 3 | Hit stagger — plays once |
| `dead` | 6 | Death — plays once, holds last frame |

### Required states for hunters (additional)

| State name | Frames | Notes |
|------------|--------|-------|
| `attack_light_1` | 4 | First hit in combo |
| `attack_light_2` | 4 | Second hit in combo |
| `attack_light_3` | 5 | Third hit — finisher |
| `attack_heavy` | 6 | Heavy attack — plays once |
| `spell_minor` | 5 | Minor spell cast |
| `spell_advanced` | 6 | Advanced spell cast |
| `spell_ultimate` | 8 | Ultimate — plays once |

### Required states for enemies (additional)

| State name | Frames | Notes |
|------------|--------|-------|
| `telegraph` | 4 | Attack wind-up — see ENEMIES.md |
| `attack` | 4 | Active hit frames |
| `recover` | 3 | Post-attack recovery |
| `wait` | 4 | Token-denied idle threat |

## SpriteAnimator.js — How It Works

`src/visuals/SpriteAnimator.js` drives UV frame stepping on a `MeshBasicMaterial` with a `map` pointing at the atlas texture.

### UV calculation

For a frame at `{ x, y, w, h }` on an atlas of size `{ W, H }`:

```js
// Offset = bottom-left of frame in UV space
offset.x = x / W;
offset.y = 1 - (y + h) / H;   // Three.js UV origin is bottom-left

// Repeat = frame size in UV space
repeat.x = w / W;
repeat.y = h / H;

material.map.offset.set(offset.x, offset.y);
material.map.repeat.set(repeat.x, repeat.y);
material.map.needsUpdate = false;  // offset/repeat changes don't need needsUpdate
```

> **Important:** Set `texture.wrapS = texture.wrapT = THREE.ClampToEdgeWrapping` when loading. Never use repeat wrapping on an atlas.

### SpriteAnimator API

```js
// Instantiate
const anim = new SpriteAnimator(material, atlasData);
// atlasData = parsed JSON from dabik.json (the frames object + meta)

// Play a state
anim.play('run');                    // loops by default
anim.play('attack_light_1', false);  // false = no loop, plays once
anim.play('dodge', false, () => {    // optional onComplete callback
  stateMachine.transition('idle');
});

// Tick every frame — pass delta time in seconds
anim.update(dt);

// Query
anim.isPlaying('idle');   // true/false
anim.currentState;        // string
anim.currentFrame;        // number
```

### Frame timing

Default: **8 fps** for all states. Override per state:

```js
const FPS_OVERRIDES = {
  idle: 6,
  run: 10,
  attack_light_1: 14,
  attack_light_2: 14,
  attack_light_3: 14,
  attack_heavy: 10,
  hurt: 18,
  dodge: 14,
  spell_ultimate: 8,
};
```

## HunterMeshes.js — Building a Hunter Sprite

`src/visuals/HunterMeshes.js` creates the Three.js mesh for each hunter.

```js
import * as THREE from 'three';

export function createHunterMesh(atlasTexture, atlasData) {
  // World size: hunters are 1.5 world units tall
  const SPRITE_HEIGHT = 1.5;
  const frame0 = Object.values(atlasData.frames)[0].sourceSize;
  const aspectRatio = frame0.w / frame0.h;
  const SPRITE_WIDTH = SPRITE_HEIGHT * aspectRatio;

  const geo = new THREE.PlaneGeometry(SPRITE_WIDTH, SPRITE_HEIGHT);

  // Pivot correction: shift geometry so bottom edge is at Y=0
  // (pivot is bottom-centre, position is at feet)
  geo.translate(0, SPRITE_HEIGHT / 2, 0);

  const mat = new THREE.MeshBasicMaterial({
    map: atlasTexture,
    transparent: true,
    alphaTest: 0.1,
    side: THREE.FrontSide,
  });

  // Clone the texture so each sprite has independent UV offset/repeat
  mat.map = atlasTexture.clone();
  mat.map.needsUpdate = true;

  const mesh = new THREE.Mesh(geo, mat);
  mesh.renderOrder = 0;  // updated each frame via Y-sort
  return mesh;
}
```

### World sizes

| Character type | Height (world units) |
|----------------|---------------------|
| Hunter | 1.5 |
| Grunt | 1.2 |
| Ranged Unit | 1.1 |
| Bruiser | 2.0 |
| Boss | 3.0 |

## Instanced Sprites (Enemies)

For enemies that appear in groups (Grunts), use `THREE.InstancedMesh` for a single draw call.

```js
// Create instanced mesh for up to 10 grunts
const geo = new THREE.PlaneGeometry(GRUNT_W, GRUNT_H);
geo.translate(0, GRUNT_H / 2, 0);
const mat = new THREE.MeshBasicMaterial({ map: gruntTexture, transparent: true, alphaTest: 0.1 });
const instancedMesh = new THREE.InstancedMesh(geo, mat, 10);

// Per-instance: set position via dummy object
const dummy = new THREE.Object3D();
dummy.position.set(x, y, -y * 0.01);  // Y-sort via Z
dummy.updateMatrix();
instancedMesh.setMatrixAt(index, dummy.matrix);
instancedMesh.instanceMatrix.needsUpdate = true;
```

> **Limitation:** All instances of an `InstancedMesh` share the same material UV. Instanced sprites therefore **cannot have per-instance animation frames**. Use instanced meshes for enemies in the same animation state (e.g., all running Grunts in the same frame), and switch to individual meshes when enemies diverge into different states (attacking, hurt, dying).

## Drop Shadow

Every character has a simple drop shadow — a dark semi-transparent ellipse quad at `y = 0` (ground level), slightly above the floor geometry.

```js
const shadowGeo = new THREE.PlaneGeometry(0.8, 0.25);
const shadowMat = new THREE.MeshBasicMaterial({
  color: 0x000000,
  transparent: true,
  opacity: 0.35,
  depthWrite: false,
});
const shadow = new THREE.Mesh(shadowGeo, shadowMat);
shadow.position.set(0, 0.01, 0);  // just above floor
shadow.renderOrder = -1;
```

Scale the shadow ellipse width with the character's `PlaneGeometry` width.

## Flipping (Facing Direction)

Flip the sprite to face left/right by scaling X:

```js
// Facing right (default)
mesh.scale.x = 1;

// Facing left
mesh.scale.x = -1;
```

Never rotate the mesh — always use scale.x flip only.

## Loading Atlas at Runtime

```js
import * as THREE from 'three';

export async function loadAtlas(name) {
  // Load JSON
  const res = await fetch(`assets/sprites/${name}.json`);
  const atlasData = await res.json();

  // Load texture
  const texture = await new Promise((resolve) => {
    new THREE.TextureLoader().load(`assets/sprites/${name}.png`, resolve);
  });

  texture.magFilter = THREE.NearestFilter;  // pixel-crisp, no blurring
  texture.minFilter = THREE.NearestFilter;
  texture.wrapS = THREE.ClampToEdgeWrapping;
  texture.wrapT = THREE.ClampToEdgeWrapping;
  texture.flipY = false;  // TexturePacker JSON coordinates are top-left origin

  return { texture, atlasData };
}
```

> Use `THREE.NearestFilter` always — these are pixel-art / clean-edge sprites. Linear filtering will blur them.

## Adding a New Character — Checklist

1. Add PNG atlas and JSON to `assets/sprites/`
2. Verify all required animation states are present (see state table above)
3. Import and call `loadAtlas('name')` in the relevant controller
4. Call `createHunterMesh()` (or equivalent enemy mesh builder) with the loaded texture + data
5. Instantiate `SpriteAnimator` with the mesh's material and the atlas data
6. Call `anim.update(dt)` in the game loop
7. Set `mesh.position` from world coords and `mesh.renderOrder = worldY * 10` each frame
8. Add drop shadow mesh as a sibling object
